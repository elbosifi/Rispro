# Neumorphism Design System Integration Plan

## Implementation Decision Summary

✅ **Scope**: Full system migration  
✅ **Accent Color**: Keep existing safety orange `#ff4757` (preserve medical RIS identity)  
✅ **Typography**: Update to Plus Jakarta Sans + DM Sans  
✅ **Strategy**: Big bang approach (update all components at once)

---

## Current System Analysis

### Tech Stack
- **Framework**: React 19 + TypeScript
- **Build Tool**: Vite 8
- **Styling**: Tailwind CSS v4 with custom CSS variables
- **Routing**: React Router DOM v7
- **State**: TanStack React Query v5
- **Icons**: Lucide React

### Existing Design System
The project already implements an **Industrial Skeuomorphism / Neumorphic** design system with:
- Base background: `#e0e5ec` (matches new system exactly!)
- Existing shadow system (card, floating, pressed, recessed states)
- Custom component classes (buttons, cards, inputs, badges)
- Matte plastic noise texture overlay
- Physical button press animations
- LED status indicators

### Key Similarities with New Design System
✅ **Base background color is identical**: `#E0E5EC`
✅ **Same neumorphic dual-shadow approach**
✅ **No borders philosophy**
✅ **Physical interaction patterns** (lift on hover, press on active)

### Key Differences to Address
1. **Shadow implementation**: New system uses RGBA transparency vs existing hex shadows
2. **Radius scale**: New system uses larger radii (32px containers, 16px buttons)
3. **Typography**: New system specifies Plus Jakarta Sans + DM Sans vs existing Inter
4. **Animation timing**: New system uses 300ms transitions vs existing 150ms
5. **Shadow depths**: New system defines additional states (Extruded Hover, Inset Deep)

---

## Detailed Implementation Plan

### Phase 1: Core Design Tokens Update (`frontend/src/index.css`)

#### 1.1 Shadow System Upgrade
Update all shadows to use RGBA transparency for smoother blending:
```css
/* New shadow system */
--shadow-card: 9px 9px 16px rgb(163,177,198,0.6), -9px -9px 16px rgba(255,255,255,0.5);
--shadow-floating: 12px 12px 20px rgb(163,177,198,0.7), -12px -12px 20px rgba(255,255,255,0.6);
--shadow-pressed: inset 6px 6px 10px rgb(163,177,198,0.6), inset -6px -6px 10px rgba(255,255,255,0.5);
--shadow-recessed: inset 10px 10px 20px rgb(163,177,198,0.7), inset -10px -10px 20px rgba(255,255,255,0.6);
--shadow-small: 5px 5px 10px rgb(163,177,198,0.6), -5px -5px 10px rgba(255,255,255,0.5);
--shadow-pressed-small: inset 3px 3px 6px rgb(163,177,198,0.6), inset -3px -3px 6px rgba(255,255,255,0.5);
```

#### 1.2 Radius Scale Update
```css
--radius-sm: 12px;
--radius-md: 16px;    /* New base button radius */
--radius-lg: 24px;
--radius-xl: 32px;    /* New card/container radius */
--radius-2xl: 32px;
--radius-full: 9999px;
```

#### 1.3 Typography Update
```css
/* Add Google Fonts import */
@import url('https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;500;700&family=Plus+Jakarta+Sans:wght@500;600;700;800&display=swap');

:root {
  --font-display: 'Plus Jakarta Sans', sans-serif;
  --font-body: 'DM Sans', sans-serif;
}

html {
  font-family: var(--font-body);
}

.font-display {
  font-family: var(--font-display);
}
```

#### 1.4 Animation Timing
Update all transitions to use 300ms duration with `ease-out` timing function.

### Phase 2: Component Refactoring

#### 2.1 Buttons
- Update border-radius from `var(--radius-lg)` to `var(--radius-md)` (16px)
- Update box-shadow to new RGBA values
- Update transitions to 300ms
- Adjust hover/active states:
  - Hover: `translateY(-1px)` + enhanced shadow
  - Active: `translateY(0.5px)` + inset shadow

#### 2.2 Cards
- Update border-radius from `var(--radius-lg)` to `var(--radius-xl)` (32px)
- Update box-shadow to new RGBA values
- Update hover transition to `translateY(-2px)`
- Keep corner screw decorative elements (unique industrial signature)

#### 2.3 Inputs
- Update border-radius from `var(--radius-md)` to `var(--radius-md)` (16px)
- Update default state to use new `--shadow-pressed`
- Update focus state to use `--shadow-recessed` (deep inset)
- Add accent ring with 2px offset

#### 2.4 Icon Housing
- Update to use inset deep shadows for "drilled" effect
- Ensure minimum 44x44px touch targets

---

### Phase 3: Layout & Responsive Updates

#### 3.1 Spacing System
- Increase vertical spacing for sections: `py-32` for hero sections
- Increase grid gaps: `gap-12` for card grids
- Maintain mobile-first approach

#### 3.2 Responsive Breakpoints
- Align with design system: `md:` (768px), `lg:` (1024px)
- Ensure font sizes scale appropriately
- Maintain neumorphic shadow effect on all screen sizes

---

### Phase 4: Accessibility & Polish

#### 4.1 Focus States
- Ensure all interactive elements have visible 2px accent rings
- Add 2px offset with background color for better visibility

#### 4.2 Contrast Verification
- Confirm primary text `#3D4852` maintains 7.5:1 contrast
- Confirm secondary text `#6B7280` maintains 4.6:1 contrast

#### 4.3 Touch Targets
- Ensure all interactive elements meet 44x44px minimum

---

### Phase 5: Micro-interactions

#### 5.1 Button Interactions
- 300ms ease-out transitions
- Subtle lift on hover, slight press on active

#### 5.2 Card Interactions
- Smooth elevation changes on hover
- Maintain physical feel

#### 5.3 Decorative Elements
- Add floating animations for ambient motion where appropriate
